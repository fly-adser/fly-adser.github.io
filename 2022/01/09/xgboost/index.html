XGBoost--A Scalable Tree Boosting System

发表于 2022-01-09 分类于 LTR预估

论文名称：XGBoost：A Scalable Tree Boosting System

XGBoost作为树模型的集大成者，在许多场景下都能够取得不错的效果。在平时的工作中也会经常用到，最近有时间对此总结一下。首先总结一下XGBoost的优缺点，然后对论文原文进行详细介绍

XGBoost优缺点总结

优点

1）在目标函数中显示地加入了正则项，用于控制模型的复杂度
正则项通过平滑叶子节点数量和叶子节点权重和来控制模型的复杂度

2）相对于gbdt使用目标函数的一阶梯度，xgboost使用了目标函数的二阶梯度
主要作用有两个：一个是支持自定义损失函数；另一个是二阶信息本身能让梯度收敛更快更准确，可以简单认为一阶导指引梯度方向，二阶导指引梯度方向如何变化

3）支持Sharinkage和列采样 Sharinkage在每次迭代后都会添加一个权重，类似学习率。它减少了每棵树的影响力，为未来的树留下空间来改进模型

4）加权直方图算法寻找最佳分割点
精确贪婪算法枚举所有可能的分割点。然而，当数据不能完全装入内存时，就无法有效地这么做。近似算法首先提出了根据特征分布的百分位数选取分裂点的方法。然后，该算法将连续特征映射到候选点划分的桶中，对统计量进行聚合，根据聚合后的统计量找到最佳分割点
然而，我们要均分的是loss，而不是样本的数量，而每个样本对loss的贡献可能是不一样的，按样本均分会导致loss分布不均匀，取到的分位点会有偏差。所以要在每个样本前面加个权重，我们把每个样本的二阶导作为其权重，就是加权直方图算法

5）稀疏感知分裂算法
在遍历特征值寻找分割点时，把缺失值分别放在左边和右边，分别计算一次收益；选取收益最大的方向作为缺失值的默认方向

6）支持并行的块结构
块结构中每一列都是排好序的特征值，它含有指向对应样本的指针，用于获取相应的一阶梯度和二阶梯度信息
在精确贪婪算法中，我们将整个数据集存储在单个块中，并通过线性扫描预先排序的特征值来运行分割搜索算法。不同特征上的分割点可并行计算，因此，一次扫描即可完成所有候选分割点的计算
块结构在使用近似算法时也有帮助。在这种情况下可以使用多个块，每个块对应于数据集中的一个行子集。不同的块可以分布在不同的机器上，或者存储在外部磁盘上，它们之间可以并行统计候选分割点的一阶梯度和二阶梯度信息。此外，列块结构还支持列抽样，因为很容易在一个块中选择列的子集

7）缓存感知访问
虽然提出的块结构有助优化分裂查找的计算复杂度，但新算法需要通过行索引间接获取梯度统计量，这是非连续的内存访问
对于贪婪算法，可以通过缓存感知的预取算法来缓解这个问题。具体来说，在每个线程中分配一个内部缓冲区，将梯度统计数据获取到其中，然后以小型批处理的方式进行累加。这种读取将直接读/写依赖更改为更长的依赖，在行数较大时，有助于降低运行时开销
对于近似算法，通过选择合适的块大小来解决这个问题。选择过小的块会导致每个线程的工作负载较小，并导致低效的并行化。但过大的块可能会导致缓存丢失，导致部分梯度统计信息无法命中缓存

8）核外计算优化 为了实现核外计算，将数据分成多个块，并将每个块存储在磁盘上。在计算过程中，使用一个独立的线程将块预取到主内存缓冲区，这样计算就可以与磁盘读取同时进行。然而，磁盘读取需要花费较长时间。因此，减少开销和增加磁盘IO的吞吐量是很重要的。我们使用两种技术来改进核外计算:

  • 块压缩：块按列压缩，并在加载到主存时由独立线程动态地解压缩
  • 块分片：第二种技术是以另一种方式将数据分割到多个磁盘上。每个磁盘分配一个预取线程，负责将数据读取到内存缓冲区中，训练线程交替地从不同缓冲区中读取数据。当有多个磁盘可用时，这有助于提高磁盘的吞吐量

缺点

  • 难以适应在线学习模式。树模型通过寻找最佳分裂点来生长树，过程是不可微的，在线模式下更新树模型是非常困难的
  • 不能很好地处理稀疏分类特征。该类特征的信息增益非常小，不能有效地利用来生长树

最佳分割点寻找方法

精确贪婪算法

树学习的关键问题之一是找到等式(7)描述最佳分割点。为了找到最佳分割点，分割查找算法在所有特征上枚举所有可能的分割，这种方式叫做精确贪婪算法。现在大多数提升树，如scikit-learn，xgboost的单机版本都支持精确贪婪算法。算法1描述了精确贪婪算法。为了有效地枚举连续特征上的所有可能分割，算法首先根据特征值对数据进行排序，并对排序后的数据进行访问，从而累积等式(7)中的梯度统计量。

近似算法

精确贪婪算法枚举所有可能的分割点。然而，当数据不能完全装入内存时，就无法有效地这么做。在分布式环境中也存在同样的问题。为了在这两种情况下有效地支持梯度提升树，需要一个近似算法
近似算法首先提出了根据特征分布的百分位数选取分裂点的方法(3.3节将会给出具体的标准)。然后，该算法将连续特征映射到候选点划分的桶中，对统计量进行聚合，根据聚合后的统计量找到最佳分割点
根据给出候选分割点的时机，该算法有两种变体。全局分割方法在树投在的初始阶段就提出所有的候选分割点，并在所有层次上使用想用的候选分裂点。局部分割方法在每次分裂后重新提出候选分割点。全局方法比局部方法需要的步骤更少。然而，对于全局方分割方法，通常需要更多的候选分裂点，因为候选分裂点在每次分裂后都没有被细化。局部分割方法在每次分割后细化候选分裂点，更适合于较深的树。图(3)给出了不同算法在同一数据集上的比较。我们发现局部分割方法确实需要更少的候选分裂点。如果有足够多的候选分裂点，全局分割方法可以和局部分割方法达到一样的准确度

大多数现有的分布式树学习近似算法也遵循这个框架。值得注意的是，特可以直接构造梯度统计量的近似直方图，或者使用其它分类策略的变体来代替分位数。分位数策略的好处是可分布式和重新计算，这一块儿将在下一小节中详细介绍。从图(3)中可以知道，在合理的近似水平下，分位数策略可以获得与精确贪婪策略相同的精度
我们的系统支持在单机场景下使用精确贪婪策略，在所有场景下使用近似算法，包含采用全局分割策略和局部分割策略的近似算法。用户可根据自己的需要自由选择

系统设计

为并行训练而设计的特征块

树学习中最耗时的部分是对数据进行排序。为了降低排序的成本，我们建议将数据存储在内存单元中，我们称之为块。每个块中的数据以压缩列(CSC)格式存储，每一列按对应的特征值进行排序。输入数据的分布只需要在训练之前计算一次，在迭代中可以重用
在精确贪婪算法中，我们将整个数据集存储在单个块中，并通过线性扫描预先排序的特征值来运行分割搜索算法。我们对所有叶子进行整体的分割查找，因此对块进行一次扫描将收集所有候选分割点的统计信息。图(6)展示了如何将数据集转换为我们需要的格式，并使用块结构找到最佳分割点
块结构在使用近似算法时也有帮助。在这种情况下可以使用多个块，每个块对应于数据集中的一个行子集。不同的块可以分布在不同的机器上，或者存储在外部磁盘上。使用排序结构，分位数查找变成了线性扫描。这对局部分割算法尤其有利，因为它在每个节点上都产生候选分割点。此外，列块结构还支持列抽样，因为很容易在一个块中选择列的子集

缓存感知访问

虽然提出的块结构有助优化分裂查找的计算复杂度，但新算法需要通过行索引间接获取梯度统计量，这是非连续的内存访问。分割枚举的简单实现是在累加和非连续内存获取操作之间引入立即读/写依赖，如图(8)所示。当梯度信息没有命中CPU缓存或者缓存丢失时，会降低查找速度
对于贪婪算法，我们可以通过缓存感知的预取算法来缓解这个问题。具体来说，我们在每个线程中分配一个内部缓冲区，将梯度统计数据获取到其中，然后以小型批处理的方式进行累加。这种读取将直接读/写依赖更改为更长的依赖，在行数较大时，有助于降低运行时开销。当数据集较大时，基于缓存感知的精确贪婪算法实现运行速度是朴素版本的两倍
对于近似算法，我们通过选择合适的块大小来解决这个问题。我们将块大小定义为一个块中包含的最大样本数，因为这反映了梯度统计的缓存存储成本。选择过小的块会导致每个线程的工作负载较小，并导致低效的并行化。但过大的块可能会导致缓存丢失，导致部分梯度统计信息无法命中缓存。块大小的合理选择需要平衡这两方面因素，实验表明，每个块存储个样本是较为合适的

用于外部计算的块

我们系统的目标之一是充分利用机器资源来实现可扩展的学习。除了处理器和内存，利用磁盘空间来处理不适合在内存中处理的数据也很重要。为了实现核外计算，我们将数据分成多个块，并将每个块存储在磁盘上。在计算过程中，使用一个独立的线程将块预取到主内存缓冲区，这样计算就可以与磁盘读取同时进行。然而，这并不能完全解决问题，因为磁盘读取需要花费较长时间。减少开销和增加磁盘IO的吞吐量是很重要的。我们使用两种技术来改进核外计算：
块压缩：块按列压缩，并在加载到主存时由独立线程动态地解压缩。我们使用一种通用的压缩算法来压缩特征值。对于行索引，我们将它减去块的起始索引，并用一个16位整数存储偏移量
块分片：第二种技术是以另一种方式将数据分割到多个磁盘上。每个磁盘分配一个预取线程，负责将数据读取到内存缓冲区中，训练线程交替地从不同缓冲区中读取数据。当有多个磁盘可用时，这有助于提高磁盘的吞吐量
